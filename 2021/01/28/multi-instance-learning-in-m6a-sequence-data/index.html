We can see that the maximum length for the positive data could be 147,584, while the maximum length for the negative sample could be 405, which is far more close

The unequal length of the samples could lead to significant bias to the results;

We therefore should try to cut the positive sequence length to minimize the bias.

We notice that the 80% sample in this virus have a roughly similar width with the positive data, so in the following sample, I would still insist on this

Prepare to do the loop

Noted that:

  • Considering the wide range of the peak width, which might lead to the calculation inefficiency, I therefore do not incorporate the padding technique to fill the gap by zero to a fixed length.

  • But we need to ensure the length between positive and negative sample to be the same.

  • That is the reason I made a lot of effort to do the following loop, to obtain the different positive and negative samples, and their length are matched accordingly.

Prepare the processed positive data (valid_pos)
  1. Ensure the processed positive data sample are smaller than the maximum length of negative sample
  2. Sort the value in decreasing order
Prepare the processed negative data

Enter the for loop

Goal: find the valid negative sample

  • Obtain the unique width of sorted valid positive data, and use this as the loop variable ==w==
  • Find the temporary positive (tmp_pos) and negative samples (tmp_neg) based on the selected width ==w== in previous step
  • Return the selected number of tmp_pos and tmp_neg (num_pos/num_neg)

Two situation after the selected temporary positive and negative sample

  1. Temporary negative sample >> tmp_pos (negative sample size is larger, resize the negative sample directly) (neg>>pos)
  • Return the names of selected temporary negative sample (neg1, neg2…)
  • Use the returned name to compare with the names in exclude names
  • Sample the names selected and return the picked names based on the number of temporary samples
  • Add the picked names to the “exclude names”
  • The next for loop is simply to extract the picked name sample and append to the final valid_neg sample
  1. pos>>neg
    • We have to increase the number of negative data.
    • The negative data could be obtained via cutting other longer sequence
  • Append all the current temporary negative sample to the final valid negative sample
  • Add to the exclude names, to update the exclude_names variable
  • Find the number of sample difference between the tmp_pos & tmp_neg and assign to num_res
  • Find the original negative sample ahead (not tmp_neg), with the width larger than in this loop ==w==
  • Same with the previous step, check whether the name is in the exclude names and select the picked names and add to the exclude names
  • the most important concepts are stated as follows (a for loop in picked names):
    • I need to find the longer negative sequence to cut down segments and attached to the temporary negative samples
    • Find the matched neg_data from the picked names called tmp_gr
    • Calculate the width difference (width_diff) between the tmp_gr from original he neg_data and ==w== in the loop.
    • Use sample function to randomly select a point, and consider this as the start and remove part of the sequence in the longer sequence
    • Similarly, to minimize the influence, we would also cut another part from the behind.

Some functions:

- Views(): for extracting sequences underneath GRanges on BSgenome object.

- DNAStringSet(): for the conversion of the BSgenome views into a set of DNA strings so that we could calculate their sequence content.

- letterFrequency(): for the calculation of the proportion / frequency of nucleotides within XStringSet.

- setdiff(x,y): find the different element in vector x and vector y, but they could only retain the vector in X

- sample(x, n): X is the whole samples, and n is the number of sample to be selected

- resize(x, width, fix="start", use.names=TRUE, ...): the width of the resized or flanking regions; fix= is an anchor
