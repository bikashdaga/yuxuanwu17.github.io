Summary

  1. This python file was to read the processed data from RDS file and conduct the embedding.

  2. In this case, instance length was set to 40, and the stride was set to 5. These two parameters could be tuned to achieve better results.

keywords

Embedding

Multi-instance learning pre-achievement

Multi-instance data preparation:

  • the train_seq, test_seq and valid_seq are all in token format (1,2,3,4)
  • They need to undergo encoding and transform to the vector format

sliding window: instance_len = 40

Moving stride: instance_stride = 5

Based on the experience in WSCNN, The number of instance per bag is [(l-c)/s]+1, where c and s are two hyper-parameters

instance sequence: [n*instance_stride, n*instance_stride+instance_len]

For instance: 1st instance [0,40], 2nd instance [5,45]

Since the token format is [1,2,3,4] and the encoding method was finished by using np.eye(4). The bag should minus one to ensure the range were in [0,1,2,3].

Note4: why npy format

The . npy format is the standard binary file format in NumPy for persisting a single arbitrary NumPy array on disk. The format stores all of the shape and dtype information necessary to reconstruct the array correctly even on another machine with a different architecture.
